Modos Ascon

Tal y como especifican los autores de Ascon, esta familia de criptosistema incluye los modos de cifrado autenticado con datos asociados, modo Hash y modo Xof. Todos los modos utilizan la misma función de permutación, que es una de las principales herramientas que utilizan los diseñadores para cifrar los datos. Sin embargo, existe una diferencia en el diseño del modo AEAD y los modos Hash y Xof.

Los autores especifican estos parámetros para los modos de Ascon:

Ascon AEAD

El modo AEAD es el modo de cifrado autenticado con datos asociados. Este modo no solo cifra la información y evita que un tercero pueda verla, sino que además autentica datos asociados. Estos datos asociados pueden ser, por ejemplo, la cabecera de un correo. Este modo de operación es muy útil en situaciones en las que, además de cifrar datos, se quiere autenticar otros que no que se van a cifrar, todo en el mismo algoritmo.

Este toma las siguientes entradas:

  • Clave secreta.
  • Nonce: Conjunto de datos que debe ser único cada vez que se cifra. No es secreto y suelen ser aleatorios. Si no se utilizase un nonce, siempre que se cifrasen los mismos datos con la misma contraseña se obtendría el mismo resultado. Esto es un comportamiento indeseado. Por esto mismo, los nonce son de un solo uso.
  • Datos en claro: Los datos a cifrar. El modo AEAD asegura la privacidad, integridad y autenticidad de estos.
  • Datos asociados: Datos que no se quieren cifrar, pero sí se quieren autenticar.

  1. Inicializar el algoritmo
  2. Fase de datos asociados

En la fase de datos asociados se introducen datos pero no se obtiene una respuesta. Esto es así porque todas las posteriores salidas del algoritmo dependen de todas las entradas anteriores. De este modo, en la fase de texto plano y la de finalización, en las que sí que se devuelven datos, si se cambian los datos asociados, el resultado cambiará radicalmente.

Ascon Hash

El modo Hash es el modo de resumen. Las funciones hash o resumen son funciones que para una entrada de cualquier tamaño devuelven un resultado de longitud fija. Son muy útiles para asegurar que los datos no han sido modificados. A esta propiedad se le denomina integridad.

Estado interno

Todas las operaciones de Ascon dependen del estado interno. Por este motivo, entender cómo funciona es vital para entender Ascon.

El estado de Ascon tiene una longitud de 320 bits. Sin embargo, para aplicar operaciones se separa en 5 bloques de 64 bits (S0,S1,S2,S3,S4). De este modo, se convierte el estado interno en una especie de matriz que será utilizada por la función de permutación.

En todos los modos de Ascon, el estado interno se construye como una concatenación de los valores de la configuración

Función de permutación

La función de permutación es la función que se aplica en cada ronda. Sin ella, no se cifrarían correctamente los datos. La seguridad de todo el criptosistema depende de la seguridad de esta.

La función de permutación de Ascon consta de 3 pasos: suma de constante de ronda, sustitución y capa lineal

  • La primera operación es la suma de constante de ronda. Como su propio nombre indica, lo que se hace es sumarle una constante al estado interno.
  • En la fase de sustitución, los valores del estado interno son sustituidos por otros siguiendo una regla específica elegida cuidadosamente por los autores. Esta se aplica en las columnas del estado interno.
  • Finalmente, la capa lineal se aplica en las filas de Ascon, y su principal objetivo es "mezclar" correctamente las filas.

Como se puede observar, la fase de sustitución y la capa lineal se aplican perpendicularmente. Esto, como todas las configuraciones y constantes de las operaciones de Ascon, fue pensado para brindar la máxima seguridad y eficiencia.
